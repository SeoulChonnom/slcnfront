## 프론트엔드 프로젝트 정보구조도(IA) 생성 + UI/UX 개선 분석

### 역할(Role)

너는 시니어 프론트엔드 아키텍트이자 UX 엔지니어다. 지금부터 주어진 프론트엔드 코드베이스를 분석해 정보구조도(Information Architecture) 와 UI/UX 개선에 바로 활용 가능한 진단 리포트를 만들어라.

### 목표(Goal)

프로젝트의 화면/기능/데이터 흐름/컴포넌트 구조를 기반으로 정보구조도(사이트맵 + 화면 계층 + 사용자 플로우) 를 작성

이후 UI/UX 개선을 위해 문제 지점과 개선 우선순위까지 함께 제안

1. 먼저 프로젝트를 스캔하며 다음을 추출하라

아래 항목을 “코드에서 근거가 보이는 것만” 정리해라.

라우팅 구조: 라우트 목록, 중첩 라우트, 동적 라우트, 가드/리다이렉트, 레이아웃(공통 프레임)

화면(페이지) 목록: 각 화면의 목적, 주요 UI 섹션, 핵심 액션(CTA), 진입/이탈 조건

주요 기능 모듈: 인증/설정/검색/리스트/상세/작성/결제 등 기능 단위로 묶기

상태관리 & 데이터 흐름: 전역 상태(store), 서버 상태(query), 캐시 전략, 폼 상태, URL state

API 연동: 엔드포인트 그룹핑, 에러 처리 방식, 로딩 처리 방식, optimistic update 여부

UI 컴포넌트 구조: 공통 컴포넌트, 도메인 컴포넌트, 페이지 전용 컴포넌트 구분

권한/접근성/국제화/테마/반응형: 존재 여부와 구현 방식

2. 산출물 1: 정보구조도(IA) 생성

아래 형식으로 출력해라.

A. 사이트맵(계층 트리)

/ Home

/login …

/dashboard …

(실제 라우트 기반으로 트리 구성)

B. 화면별 요약 카드(표 형식)
각 화면(라우트)마다 아래 필드를 채워라.

Screen ID / Route

사용자 목적(1문장)

주요 콘텐츠(섹션 단위)

주요 액션(CTA)

필요 데이터(API/Store)

상태(loading/empty/error) 처리 여부

접근성/반응형/성능 주의점(있다면)

C. 사용자 플로우(최소 3개 이상)
코드상 가능한 대표 플로우를 뽑아 순서도로 적어라. 예:

“비로그인 → 로그인 → 메인 진입”

“리스트 → 필터 → 상세 → 액션(저장/공유/삭제)”

“에러 발생 → 재시도/대체 경로”

가능하면 Mermaid 로도 제공하라. (flowchart 또는 journey)

3. 산출물 2: UI/UX 개선 관점 진단 리포트

정보구조도를 바탕으로 아래를 작성하라.

A. IA 관점 문제 후보

중복/유사 화면 존재 여부

라우트/메뉴 네이밍 일관성

깊은 단계(클릭 4단계 이상) / 사용자가 길 잃기 쉬운 구간

핵심 기능으로 가는 경로가 복잡한 지점

B. 화면 상태 UX 품질 점검 체크리스트 결과
각 페이지에서 아래 항목이 구현되어 있는지 “있음/부분/없음”으로 표시하고 근거 파일/컴포넌트를 인용하라.

로딩 스켈레톤/로딩 인디케이터

empty state(데이터 없음) 안내

에러 메시지(원인/행동 제안/재시도)

폼 검증(실시간/제출 시), 포커스 이동

토스트/알림/Undo

접근성(키보드 탐색, aria, 대비)

모바일/반응형 레이아웃

C. 개선 우선순위(ROI 기준) Top 10

(1) 문제 요약

(2) 사용자 영향(빈도/치명도)

(3) 수정 난이도(낮/중/높)

(4) 추천 해결책(구체적으로: 어떤 컴포넌트/패턴으로 바꿀지)

(5) 관련 코드 위치(파일/폴더 추정 또는 실제 근거)

4. 출력 규칙

가능한 한 구체적으로: 라우트/파일/컴포넌트 이름을 인용

모르면 추측하지 말고 “확인 불가(근거 없음)”라고 표시

결과는 Markdown으로 깔끔하게 구성

5. 추가 컨텍스트

- UI/UX 개선 목적: 현재 프로젝트의 디자인이 너무 촌스럽고, 사용성이 떨어져서 전면 개편을 원함.
- 모든 산출물은 한글로 작성하고, docs/ 폴더에 저장할 것.
